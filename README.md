# software-testing-quality

Репозиторій **software-testing-quality** містить дві програмні системи, розроблені в межах дослідження якості процесу тестування програмного забезпечення. Обидві програми призначені для підтримки прийняття рішень у тестовому процесі та автоматизації аналітичних задач.

---

## 1. Програма розрахунку вагових пріоритетів факторів  
**Метод:** бінарні (парні) порівняння

### Опис
Перша програма призначена для розрахунку вагових коефіцієнтів факторів якості процесу тестування на основі експертних оцінок.  
Для цього використовується **метод бінарних (парних) порівнянь**, що дозволяє:

- формувати матрицю експертних оцінок;
- визначати відносну важливість кожного фактора;
- обчислювати нормовані вагові коефіцієнти;
- перевіряти узгодженість експертних суджень.

### Основні можливості
- Введення матриці парних порівнянь факторів.
- Автоматичний розрахунок:
  - власних векторів пріоритетів;
  - максимального власного значення (λmax);
  - індексу узгодженості (ІП);
  - відношення узгодженості (ВП).
- Відображення проміжних та підсумкових результатів.

### Призначення
Програма використовується для:
- визначення ваг факторів якості тестового процесу;
- підтримки експертного оцінювання;
- формування інтегральних показників якості.

---

## 2. Програма пошуку дублікатів звітів про помилки  
**Метод:** семантичний пошук + класифікація

### 2.1 Етап підготовки системи
На цьому етапі відбувається підготовка моделі для виявлення дублікатів:

- завантаження набору даних із звітами про помилки;
- вибір моделі пошуку та класифікації;
- навчання класифікатора;
- ініціалізація системи перед використанням.

Система формує внутрішнє представлення звітів та готує модель до подальшого пошуку.

---

### 2.2 Етап пошуку дублікатів
Після підготовки системи користувач може виконувати пошук дублікатів:

- введення нового звіту про помилку;
- автоматичний аналіз тексту звіту;
- пошук найбільш подібних існуючих звітів;
- відображення знайдених дублікатів.

### Основні можливості
- Семантичний пошук подібних звітів.
- Автоматичне визначення потенційних дублікатів.
- Підтримка різних моделей пошуку та класифікації.
- Інтерактивний графічний інтерфейс.

### Призначення
Програма використовується для:
- зменшення кількості дубльованих звітів у баг-трекінгових системах;
- прискорення обробки нових помилок;
- підвищення ефективності тестового процесу.
